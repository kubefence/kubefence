  <action>
Create `Dockerfile` at repo root with a multi-stage build:

**Stage 1 — Builder (`golang:1.24-alpine`):**
```dockerfile
FROM golang:1.24-alpine AS builder
WORKDIR /workspace
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o 10-nono-nri ./cmd/nono-nri
```

**Stage 2 — Runtime (`alpine:3.20`):**
```dockerfile
FROM alpine:3.20
RUN apk add --no-cache ca-certificates
COPY --from=builder /workspace/10-nono-nri /usr/local/bin/10-nono-nri
# The nono binary must be present in the build context.
# Download or copy it to ./nono before running docker build.
# See: make docker-build
COPY nono /usr/local/bin/nono
ENTRYPOINT ["/usr/local/bin/10-nono-nri"]
```

Use `alpine:3.20` instead of `FROM scratch` because the init container needs `cp` and `sh` available in the image (scratch has no utilities). This adds ~7MB but enables the DaemonSet init container `cp` command to work directly.

Binary paths in the image (`/usr/local/bin/10-nono-nri` and `/usr/local/bin/nono`) MUST match the DaemonSet container command and init container cp source from `deploy/daemonset.yaml`.

Add a `.dockerignore` file to exclude unnecessary files from build context:
```
.planning/
.git/
*.md
!go.mod
!go.sum
deploy/
10-nono-nri
```

Add the following to the existing Makefile, preserving all existing targets. Append after the existing `lint` target:

```makefile
IMAGE ?= nono-nri:latest
KIND_CLUSTER ?= nono-test

.PHONY: docker-build docker-load-kind

docker-build:
	@test -f nono || (echo "ERROR: ./nono binary not found in build context. Download from nono releases and place at repo root." && exit 1)
	docker build -t $(IMAGE) .

docker-load-kind: docker-build
	kind load docker-image $(IMAGE) --name $(KIND_CLUSTER)
```

The `docker-build` target:
- Checks that `./nono` binary exists in the build context (fails with clear error if missing)
- Builds the Docker image tagged as `nono-nri:latest` (configurable via `IMAGE` variable)

The `docker-load-kind` target:
- Depends on `docker-build` (builds first)
- Loads the image into a Kind cluster named `nono-test` (configurable via `KIND_CLUSTER`)

Add `docker-build` and `docker-load-kind` to the `.PHONY` list.
